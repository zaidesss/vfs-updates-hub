📝 SECTION 7: CREATING UPDATES
═══════════════════════════════════════════════════════════════════════════════

This is one of your PRIMARY admin functions.

─────────────────────────────────────────────────────────────────────────────
HOW TO CREATE A NEW UPDATE
─────────────────────────────────────────────────────────────────────────────

STEP 1: Go to Admin → Admin Panel
STEP 2: Click on the "Updates" tab
STEP 3: Click "Create Update" or "New Update" button
STEP 4: Fill in the form:

REQUIRED FIELDS:
• Title - Clear, descriptive title
• Category - Select from dropdown:
  - Orders & Transactions
  - Payments & Billing
  - Shipping & Tracking
  - Delivery Issues
  - International & Customs
  - Product Issues
  - Product Information
  - Subscriptions
  - Warehouse & Fulfillment
  - Internal Operations
• Summary - Brief overview (shows in card view)
• Body - Full content (supports markdown)

OPTIONAL FIELDS:
• Help Center URL - Link to related article
• Deadline - Date by which agents must acknowledge
• Status - Draft (not visible) or Published (visible to all)

STEP 5: Preview your content
STEP 6: Click "Save" or "Publish"

─────────────────────────────────────────────────────────────────────────────
USING THE MARKDOWN EDITOR
─────────────────────────────────────────────────────────────────────────────

The body field supports Markdown formatting:

BASIC FORMATTING:
• **bold text** → bold text
• *italic text* → italic text
• # Heading 1
• ## Heading 2
• ### Heading 3

LISTS:
• - Item 1 (bulleted list)
• 1. Item 1 (numbered list)

LINKS:
• [Link Text](URL)

IMPORTANT NOTES:
• You can preview before publishing
• Editor shows formatting in real-time
• Keep formatting consistent

─────────────────────────────────────────────────────────────────────────────
UPDATE STATUSES EXPLAINED
─────────────────────────────────────────────────────────────────────────────

DRAFT:
• Not visible to regular users
• Use for work-in-progress
• Publish when ready

PUBLISHED:
• Visible to all users
• Appears in Updates page
• Agents can acknowledge
• Questions can be asked

OBSOLETE:
• Marked as outdated
• May still be visible in Knowledge Base
• No longer requires acknowledgement

─────────────────────────────────────────────────────────────────────────────
SIMILAR UPDATES CHECK (AI Feature)
─────────────────────────────────────────────────────────────────────────────

Before creating a new update:

STEP 1: Click "Check Similar Updates"
STEP 2: AI analyzes existing updates
STEP 3: See similarity scores:
        • HIGH similarity - Very similar exists
        • MEDIUM similarity - Somewhat related
        • LOW similarity - Different topic

USE THIS TO:
• Avoid duplicate content
• Link related updates
• Update existing content instead

✏️ SECTION 8: EDITING UPDATES
═══════════════════════════════════════════════════════════════════════════════

─────────────────────────────────────────────────────────────────────────────
HOW TO EDIT AN EXISTING UPDATE
─────────────────────────────────────────────────────────────────────────────

STEP 1: Go to Admin → Admin Panel → Updates tab
STEP 2: Find the update in the list
STEP 3: Click "Edit" button in Actions column
STEP 4: Modify any fields
STEP 5: Click "Save" to apply changes

WHAT CAN BE EDITED:
• Title
• Category
• Summary
• Body content
• Help Center URL
• Deadline
• Status

─────────────────────────────────────────────────────────────────────────────
CHANGE HISTORY
─────────────────────────────────────────────────────────────────────────────

All changes are tracked:
• Who made the change
• When it was made
• What was changed
• Previous values

This creates an audit trail for compliance.

─────────────────────────────────────────────────────────────────────────────
DELETING UPDATES
─────────────────────────────────────────────────────────────────────────────

STEP 1: Find the update in Admin Panel
STEP 2: Click "Delete" button
STEP 3: Confirm the deletion

⚠️ WARNING: Deletion is PERMANENT
• Acknowledgements are lost
• Questions are lost
• This cannot be undone
• Consider using "Obsolete" status instead

❓ SECTION 9: MANAGING QUESTIONS
═══════════════════════════════════════════════════════════════════════════════

Agents can ask questions on any update. You manage these.

─────────────────────────────────────────────────────────────────────────────
VIEWING ALL QUESTIONS
─────────────────────────────────────────────────────────────────────────────

STEP 1: Go to Admin → Admin Panel
STEP 2: Click "Questions" tab
STEP 3: See all questions listed

INFORMATION SHOWN:
• Reference number
• Update the question is about
• Question text
• Who asked (agent email)
• When asked
• Status (Pending/Replied)

─────────────────────────────────────────────────────────────────────────────
REPLYING TO QUESTIONS
─────────────────────────────────────────────────────────────────────────────

STEP 1: Click on a pending question
STEP 2: Read the full question
STEP 3: View the related update for context
STEP 4: Type your reply
STEP 5: Click "Send Reply"

AFTER REPLYING:
• Status changes to "Replied"
• Agent receives notification
• Reply is visible on the update

─────────────────────────────────────────────────────────────────────────────
THREADED DISCUSSIONS
─────────────────────────────────────────────────────────────────────────────

Questions can become threads:
• Agent asks initial question
• You reply
• Agent can follow up
• Conversation continues

BEST PRACTICES:
• Reply promptly (within 24 hours if possible)
• Be clear and complete
• Reference update content when helpful
• Escalate complex questions if needed
