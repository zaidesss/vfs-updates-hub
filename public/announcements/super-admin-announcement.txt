👥 SECTION 5: USER MANAGEMENT (SUPER ADMIN EXCLUSIVE)
═══════════════════════════════════════════════════════════════════════════════

This is your EXCLUSIVE function as Super Admin.

─────────────────────────────────────────────────────────────────────────────
CREATING NEW USERS
─────────────────────────────────────────────────────────────────────────────

STEP 1: Go to Admin → Admin Panel
STEP 2: Click on "Users" tab
STEP 3: Click "Create User" or "Add New User"
STEP 4: Fill in the form:
        • Email address (required, must be unique)
        • Name (required)
        • Role (required):
          - User (standard agent)
          - HR (HR team member)
          - Admin (administrator)
          - Super Admin (full access)
        • Set temporary password
        • "Must change password" flag (recommended: ON)
STEP 5: Click "Create"

AFTER CREATION:
• User appears in user list
• User can log in immediately
• If "must change password" is set, they'll be prompted
• Add them to agent_profiles for profile data

─────────────────────────────────────────────────────────────────────────────
ROLES EXPLAINED
─────────────────────────────────────────────────────────────────────────────

USER (Agent):
• View and acknowledge updates
• Ask questions
• Submit leave requests
• View personal data
• Standard access

HR:
• All User permissions PLUS:
• Approve/reject leave requests
• View all agent profiles
• Access outage statistics
• Limited admin panel access

ADMIN:
• All HR permissions PLUS:
• Create/edit/delete updates
• Reply to questions
• Full admin panel access
• Cannot manage users

SUPER ADMIN:
• ALL permissions
• Create/delete users
• Change user roles
• Add improvements
• System configuration

─────────────────────────────────────────────────────────────────────────────
EDITING USERS
─────────────────────────────────────────────────────────────────────────────

STEP 1: Find user in the list
STEP 2: Click "Edit"
STEP 3: Modify:
        • Name
        • Role
        • Must change password flag
STEP 4: Save changes

─────────────────────────────────────────────────────────────────────────────
CHANGING USER EMAIL
─────────────────────────────────────────────────────────────────────────────

This is a special operation:
STEP 1: Find the user
STEP 2: Click "Change Email"
STEP 3: Enter new email
STEP 4: Confirm the change

NOTE: This updates both auth and roles tables.

─────────────────────────────────────────────────────────────────────────────
DELETING USERS
─────────────────────────────────────────────────────────────────────────────

STEP 1: Find the user
STEP 2: Click "Delete"
STEP 3: Confirm deletion

⚠️ WARNINGS:
• Deletion removes user from auth system
• User data may be preserved for audit
• Deleted users can be restored
• Cannot delete the last Super Admin

─────────────────────────────────────────────────────────────────────────────
RESTORING DELETED USERS
─────────────────────────────────────────────────────────────────────────────

Deleted users are tracked:
• View "Deleted Users" section
• Click "Restore" to bring back
• User regains original role
• May need to reset password

─────────────────────────────────────────────────────────────────────────────
PROTECTION RULES
─────────────────────────────────────────────────────────────────────────────

SYSTEM PROTECTIONS:
• Cannot delete the last Super Admin
• Cannot demote the last Super Admin
• System tracks Super Admin count
• At least one Super Admin must exist

🛡️ SECTION 6: ROLE MANAGEMENT
═══════════════════════════════════════════════════════════════════════════════

─────────────────────────────────────────────────────────────────────────────
PROMOTING USERS
─────────────────────────────────────────────────────────────────────────────

To give someone elevated access:
STEP 1: Edit the user
STEP 2: Change their role dropdown
STEP 3: Save

PROMOTION PATHS:
• User → HR (give HR access)
• User → Admin (give admin access)
• HR → Admin (upgrade to full admin)
• Any role → Super Admin (give full access)

─────────────────────────────────────────────────────────────────────────────
DEMOTING USERS
─────────────────────────────────────────────────────────────────────────────

To reduce someone's access:
STEP 1: Edit the user
STEP 2: Change their role to lower level
STEP 3: Save

CONSIDERATIONS:
• Be sure this is appropriate
• User loses higher-level functions immediately
• Cannot demote last Super Admin

─────────────────────────────────────────────────────────────────────────────
VIEWING BY ROLE
─────────────────────────────────────────────────────────────────────────────

In Admin Panel:
• "Users" tab shows all users with "user" role
• "Admins" tab shows admins and super admins
• Filter options available

⚠️ SECTION 13: COMMON SUPER ADMIN SCENARIOS
═══════════════════════════════════════════════════════════════════════════════

SCENARIO: New employee joining
1. Create user account with appropriate role
2. Set "must change password" flag
3. Create agent profile
4. Send credentials securely
5. Verify they can log in

SCENARIO: Employee leaving
1. Delete or deactivate user account
2. Note deletion in records
3. User loses access immediately
4. Can restore if needed later

SCENARIO: Role change request
1. Verify the request is appropriate
2. Edit user role
3. Notify the user
4. They get new access immediately

SCENARIO: Password reset needed
1. Reset via admin tools
2. Set temporary password
3. Enable "must change password"
4. Communicate securely

SCENARIO: Someone needs Super Admin
1. Consider carefully - is it necessary?
2. Discuss responsibilities
3. Only promote if truly needed
4. Ensure backup Super Admin exists

SCENARIO: Accidentally deleted user
1. Go to deleted users section
2. Find the user
3. Click restore
4. User regains access

🔄 SECTION 14: BACKUP SUPER ADMIN
═══════════════════════════════════════════════════════════════════════════════

IMPORTANT: Always maintain at least 2 Super Admins

WHY:
• System requires at least one Super Admin
• If you're unavailable, another can act
• Prevents lockout scenarios
• Business continuity

RECOMMENDATIONS:
• Have a primary Super Admin
• Have at least one backup
• Both should know responsibilities
• Document handoff procedures
